## Suggested Learning Roadmap

### Phase 1: Foundation (Beginner)

- Networking fundamentals (OSI, TCP/IP, routing, DNS, HTTP/HTTPS).
- Linux essentials (file system, permissions, users, processes, Bash).
- Security basics (CIA triad, threats, vulnerabilities, risk).
- Basic scripting in Python and Bash.

Deliverables:

- Build a personal command cheat sheet.
- Complete at least 2 beginner rooms/labs per week.
- Write one summary note per topic learned.

### Phase 2: Offensive Basics

- Recon and enumeration (whois, nslookup, nmap).
- Web fundamentals and common web vulnerabilities.
- Authentication and session security basics.
- Intro to Burp Suite and traffic interception.

Deliverables:

- Complete beginner web security labs.
- Perform full recon workflow in a legal lab environment.
- Document findings in a mini pentest-style report.

### Phase 3: Intermediate Practice

- Privilege escalation (Linux and Windows basics).
- Active Directory fundamentals.
- Basic malware analysis and incident response awareness.
- Capture the Flag workflow and time management.

Deliverables:

- Solve medium-difficulty CTF challenges.
- Create a reusable checklist for box exploitation.
- Practice report writing with remediation steps.

### Phase 4: Career Readiness

- Specialize (web app pentesting, SOC, cloud security, red teaming).
- Build portfolio projects and publish writeups.
- Prepare for certifications and interviews.
- Participate in community discussions and competitions.

Deliverables:

- Maintain a public or private portfolio of labs and lessons.
- Create one professional pentest report sample.
- Attempt one certification after consistent preparation.

## Important Ethical Hacking Checklist

Use this as a living checklist and mark progress over time.

### Legal and Ethical Readiness

- [ ] I only test systems I own or have explicit written permission to test.
- [ ] I understand local cyber laws and responsible disclosure basics.
- [ ] I never keep or misuse sensitive data found in labs/tests.
- [ ] I stop testing immediately if engagement scope is unclear.

### Technical Fundamentals

- [ ] I understand TCP/IP, subnetting, ports, and protocols.
- [ ] I can explain DNS, HTTP, HTTPS, and TLS at a practical level.
- [ ] I am comfortable navigating Linux via terminal.
- [ ] I can write basic automation scripts in Python or Bash.

### Lab Setup and Workflow

- [ ] I have a safe lab setup (VMs, snapshots, isolated network).
- [ ] I use note-taking templates for each machine/lab.
- [ ] I save commands, outputs, and screenshots during testing.
- [ ] I track what worked and what failed after each session.

### Offensive Security Skills

- [ ] I can perform recon and service enumeration systematically.
- [ ] I can validate common web vulnerabilities in legal labs.
- [ ] I can perform basic privilege escalation checks.
- [ ] I can use Burp Suite, nmap, and other core tools effectively.

### Defensive and Professional Skills

- [ ] I can map findings to risk severity and business impact.
- [ ] I can suggest practical remediation steps.
- [ ] I can write clear, structured security reports.
- [ ] I communicate findings professionally and responsibly.

### Growth and Consistency

- [ ] I practice at least 4-6 focused sessions per week.
- [ ] I complete regular CTF/lab challenges and review mistakes.
- [ ] I revisit weak topics monthly and retest understanding.
- [ ] I update my roadmap based on real progress.

## Recommended Practice Routine (Weekly)

- 2 days: Fundamentals (networking/Linux/scripting).
- 2 days: Hands-on labs (web, enumeration, privilege escalation).
- 1 day: CTF challenge + writeup.
- 1 day: Review weak areas and update notes/checklist.
- 1 day: Rest or light revision.
